# immutable-rules 📦

> Система неизменяемых правил и управления проектами для AI агентов (OpenClaw).

Когда даёшь задачу ИИ агенту — он **обязан** следовать правилам: проверить проект, записать задачу, сделать бэкап, закоммитить. Без исключений.

## 🎯 Для чего

- **Забыть про "мы это не делали"** — каждый шаг записывается в changelog (append-only)
- **Контроль задач** — единый менеджер проектов для всех агентов
- **Сохранение контекста** — "сохрани" → дневник + задачи + Git
- **ТЗ под каждый проект** — фичи с чекбоксами, статусы, история
- **Сеть агентов** — один файл конфигурации на всех, легко масштабировать

## 📜 6 правил

| # | Правило | Суть |
|---|--------|------|
| 1 | Workflow при задаче | Проект → менеджер → бэкап → коммит → память |
| 2 | PROJECT_CHANGELOG | Append-only лог каждого действия по проекту |
| 3 | Запрет игнорировать changelog | Нельзя сказать "не делали" без проверки |
| 4 | Диагностика поломок | Changelog → Git log → откат |
| 5 | ТЗ (TZ.md) | Техническое задание под каждый проект с фичами |
| 6 | Сохранение + Git + SERVERS.md | "Сохрани" / "Пуш в git" / "Сохрани доступ" |

## 🚀 Установка

Скажи своему ИИ агенту:

```
установи immutable-rules
```

Агент сам скачает скилл с GitHub, задаст 5-6 вопросов и настроит всё.

Или вручную:

```bash
cd ~/.openclaw/workspace && \
mkdir -p skills/immutable-rules/references && \
curl -sL https://github.com/mwoodowl-byte/immutable-rules/archive/refs/heads/main.tar.gz | \
tar xz --strip-components=1 -C skills/immutable-rules/
```

## 💬 Команды

| Команда | Описание |
|---------|----------|
| `/rule` | Записать новое правило |
| "покажи правила" | Список всех правил |
| "покажи лог правил" | Лог выполнения |
| "сохрани" | Дневник + задачи |
| "сохрани в глубокую память" | MEMORY.md + Git |

## 🏗 Архитектура

```
Владелец (Telegram)
    │
    ├── Агент 1 (Сервер A) ──┐
    ├── Агент 2 (Сервер B) ──┼── VPS (центральный)
    ├── Агент 3 (Сервер C) ──┤    ├── PROJECTS_TASKS.md
    └── Агент N (Сервер N) ──┘    ├── AGENT_NETWORK.md
                                  ├── TZ.md (под каждый проект)
                                  └── Веб-версия менеджера
```

## 📁 Ключевые файлы

| Файл | Где | Кто меняет |
|------|-----|-----------|
| RULES.md | Workspace агента | Только владелец |
| RULES_LOG.md | Workspace агента | Агент (append-only) |
| CONFIG.md | Workspace агента | При установке |
| SERVERS.md | Workspace агента | Владелец |
| PROJECTS_TASKS.md | Центральный сервер | Агенты + владелец |
| PROJECT_CHANGELOG.md | Папка проекта | Агенты (append-only) |
| TZ.md | Папка проекта | Владелец + агент |
| AGENT_NETWORK.md | Центральный сервер | Владелец + агент |

## 🔗 Структура скилла

```
immutable-rules/
├── SKILL.md              — Правила + инфраструктура + установка
└── references/
    ├── HELP.md           — Справка (как добавить агента, проект)
    ├── INSTALL.md        — Полная инструкция установки
    ├── CONFIG_TEMPLATE.md — Шаблон конфигурации
    ├── TZ_TEMPLATE.md    — Шаблон Технического Задания
    ├── DEFAULT_RULES.md  — 6 дефолтных правил
    ├── onboarding.md     — Быстрый старт
    └── rule-templates.md — Шаблоны правил
```
